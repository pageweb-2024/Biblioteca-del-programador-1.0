HTML

HISTORIA DEL LENGUAJE
    ORIGENES

    HTML fue creado por Tim Berners-Lee, un científico del CERN (Organización Europea para la Investigación Nuclear). Berners-Lee creó HTML como parte de un proyecto para compartir información entre científicos a través de una red, lo que se convertiría en la World Wide Web.

    PRIMERA VERSION

    HTML Tags: Se publicó la primera versión del documento llamado "HTML Tags", que contenía 18 etiquetas básicas como:" p, h1, a, img ", etc.

    EVOLUCION
    • HTML 2.0 (1995) El IETF (Internet Engineering Task Force) creó un grupo de trabajo para estandarizar HTML y, en 1995, lanzó HTML 2.0.
    • HTML 4.01 (1999) HTML 4.01 fue una de las versiones más estables y ampliamente utilizadas durante varios años. Introdujo un soporte más extenso para la accesibilidad, formularios mejorados, y hojas de estilo en cascada (CSS) para la presentación.
    • XHTML (2000) XHTML (Extensible HyperText Markup Language) fue una reformulación de HTML 4.01 basada en XML. Se introdujo para hacer que el lenguaje fuera más estricto y compatible con XML, lo que permitiría que los documentos HTML se procesaran de manera más uniforme en diferentes plataformas.
    • HTML5 (2014) HTML5 fue una gran actualización del lenguaje, diseñada para ser más adaptable a la multimedia moderna y al desarrollo de aplicaciones web más complejas. Se introdujeron nuevas etiquetas como article, section, nav, header, y footer, lo que promovió una estructura más semántica en los documentos. HTML5 también incluyó soporte nativo para audio y video, y mejoras significativas para el manejo de formularios y la interactividad, eliminando la necesidad de complementos como Flash.
    PRESENTE

    Desde la versión de HTML5, el lenguaje sigue evolucionando de manera continua a través de mejoras incrementales, lideradas por el WHATWG. Se ha abandonado la noción de lanzar nuevas "versiones" formales y en su lugar se trabaja en un modelo de HTML vivo, que se actualiza progresivamente según las necesidades del desarrollo web moderno.

PORQUE ES IMPORTANTE

    HTML es importante por varias razones clave, ya que forma la base del desarrollo web y de cómo interactuamos con la información en Internet.

    Estructura y Semántica

    HTML proporciona una estructura clara para el contenido a través de etiquetas como header, footer, article, section, etc. Esto no solo mejora la legibilidad del código para los desarrolladores, sino que también hace que el contenido sea más accesible para diferentes dispositivos, incluidos los lectores de pantalla que utilizan personas con discapacidades visuales.

    Accesibilidad

    La semántica de HTML ayuda a que los sitios web sean accesibles para todos, incluidos aquellos con discapacidades. Al usar etiquetas HTML adecuadas, como alt para descripciones de imágenes o nav para definir áreas de navegación, se garantiza que las personas que usan tecnologías de asistencia puedan interactuar correctamente con el contenido.

    Multimedia y Aplicaciones Web Modernas

    HTML5 introdujo la capacidad de manejar de manera nativa multimedia (audio, video) y gráficos vectoriales (SVG y canvas). Esto ha permitido el desarrollo de aplicaciones web más ricas e interactivas sin depender de complementos externos como Flash, lo que mejora la seguridad, rendimiento y accesibilidad.

CONCEPTOS BASICOS DEL LENGUAJE
    Etiquetas (Tags)

    Las etiquetas son los bloques básicos de HTML. Se usan para definir diferentes partes de una página web. La mayoría de las etiquetas tienen una estructura de apertura y cierre:

    • Etiqueta de apertura: nombre_de_etiqueta.
    • Etiqueta de cierre: /nombre_de_etiqueta.
    • p es la etiqueta de apertura y /p es la etiqueta de cierre.
    • img (para imágenes) y br (para saltos de línea).
    Atributos (Attributes)

    Los atributos proporcionan información adicional sobre una etiqueta. Se colocan dentro de la etiqueta de apertura y generalmente siguen este formato: nombre_atributo="valor". src define la fuente de la imagen, y alt proporciona un texto alternativo que describe la imagen en caso de que no se cargue.

    Estructura del Documento

    Todo documento HTML tiene una estructura básica que sigue un formato específico.

    • !DOCTYPE html: Instrucción que indica al navegador que se está utilizando HTML5.
    • html: Etiqueta raíz que envuelve todo el contenido del documento HTML.
    • head: Contiene metadatos sobre el documento (por ejemplo, título, enlaces a archivos CSS, scripts).
    • title: Define el título de la página que se muestra en la pestaña del navegador.
    • body: Contiene el contenido visible de la página (texto, imágenes, videos, etc.).
    Encabezados (Headings)

    HTML tiene seis niveles de encabezados, desde h1 (el más importante) hasta h6 (el menos importante), que se utilizan para definir títulos y subtítulos en una página

    Párrafos (Paragraphs)

    El contenido textual en HTML generalmente se organiza en párrafos usando la etiqueta p.

    Enlaces (Links)

    Los enlaces en HTML se crean con la etiqueta a. El atributo más importante para un enlace es href, que especifica la URL a la que apunta.

    Listas

    HTML soporta dos tipos principales de listas: listas ordenadas y listas no ordenadas.

    • Listas ordenadas (ol): Los elementos de la lista están numerados.
    • Listas no ordenadas (ul): Los elementos de la lista están marcados son viñetas.
    Tablas (Tables)

    Las tablas en HTML se crean con la etiqueta table, y los elementos de la tabla como filas (tr), encabezados (th), y celdas (td) se definen dentro de ella.

    Formularios (Forms)

    Los formularios permiten a los usuarios enviar datos a un servidor. Se crean con la etiqueta form y contienen elementos de entrada como campos de texto (input), botones (button), y áreas de texto (textarea).

    Divs y Spans
    • div: Elemento de bloque usado para agrupar contenido. Se usa comúnmente para aplicar estilos o estructuras a secciones grandes de una página.
    • span: Elemento en línea que se usa para agrupar contenido de menor tamaño. A menudo se usa para aplicar estilos a partes de un texto.
HERRAMIENTAS CON LAS QUE SE USA
    Visual Studio Code

    Un editor de código gratuito y potente que admite una gran cantidad de extensiones para HTML, CSS, JavaScript y otros lenguajes.

    Sublime Text

    Un editor ligero y rápido, con una interfaz limpia y muchas características útiles como resaltado de sintaxis y completado automático.

    Atom

    Un editor de texto hackeable y colaborativo, desarrollado por GitHub, que también ofrece muchas extensiones.

    Notepad++

    Un editor de texto gratuito y de código abierto que es ligero y tiene soporte para múltiples lenguajes de programación.

LENGUAJES Y TECNOLOGIAS QUE COMPLEMENTAN EL LENGUAJE
    CSS

    CSS se utiliza para definir el estilo y la presentación de un documento HTML. Permite a los desarrolladores aplicar colores, fuentes, márgenes, alineaciones y otros aspectos visuales a los elementos HTML.

    JavaScript

    JavaScript es un lenguaje de programación que permite añadir interactividad y dinámismo a las páginas web. Se utiliza para manipular el DOM (Document Object Model), gestionar eventos, realizar peticiones a servidores y mucho más.

    PHP

    PHP es un lenguaje de scripting del lado del servidor que se utiliza para crear aplicaciones web dinámicas. Permite interactuar con bases de datos y generar contenido HTML de manera programática.
